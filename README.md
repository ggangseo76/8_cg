# 8_cg

컴퓨터 그래픽스 팀프로젝트: 3D 미니 전시관

## 개발 환경
- C++
- OpenGL
- GLFW
- GLAD

## 폴더 구조
- src/: 소스코드
- shaders/: 셰이더 코드
- resources/: 텍스처, 이미지 등 리소스 파일
- docs/: 보고서, 회의록, 발표자료
- record/: 시연 영상 관련 자료

## 작업 방식
1. main 브랜치는 최종본/안정본으로 사용
2. 각자 기능별 브랜치를 만들어 작업
3. 작업 후 Pull Request로 main에 병합

## 브랜치 예시
- feature/camera
- feature/lighting
- feature/objects
- feature/texture
- docs/report

## 프로젝트 구조

전시관은 크게 **사각형 갤러리**(안내방 → 계단 → 메인방), 그 옆의 **원형 플라네타륨 돔**,
그리고 돔 중앙에 떠 있는 **목성계**로 구성된다. 방문자는 1인칭 카메라로 공간을 자유롭게
걸어다니며 관람하고, 계단을 통해 높이가 다른 두 공간을 오간다.

### 주요 구성 요소 (클래스)
| 파일 | 역할 |
|------|------|
| `main.cpp` | 프로그램 진입점. 렌더 루프, 입력 처리, 충돌 판정, 씬 구성 |
| `Camera` | 1인칭 카메라. 바닥 높이 따라가기로 계단을 자연스럽게 오르내림 |
| `Room` | 직사각형 갤러리(바닥·천장·벽). 오른쪽 벽은 돔으로 가는 통로로 개방 |
| `Staircase` | 단차 구조(안내방 평지 → 계단 → 메인방). 위치별 바닥 높이 제공 |
| `PlanetariumDome` | 원형 돔(바닥·곡면 벽·천장). 입구 구간을 비워 통로로 연결 |
| `InfoPanel` | 안내판(받침대 + 기울어진 판 + 정보 텍스처). 바라보는 방향 지정 가능 |
| `JupiterSystem` | 목성과 4대 위성(이오·유로파·가니메데·칼리스토)의 자전·공전 |
| `Shader` | 셰이더 로드/관리 |

### 셰이더
| 파일 | 용도 |
|------|------|
| `room.vert` / `room.frag` | 텍스처 + 조명 (방·돔·안내판) |
| `jupiter.vert` / `jupiter.frag` | 텍스처 + 방향광 조명 (행성·위성) |
| `basic.vert` / `basic.frag` | 초기 테스트용 (현재 미사용) |

### 이동 · 충돌 처리
- **걸을 수 있는 영역** = (방 사각형 ∪ 돔 원) − 안내판 반경
- 매 프레임 카메라의 다음 위치를 검사해, 영역 밖이면 이동을 취소한다.
  X·Z 축을 분리해 검사하므로 벽에 부딪혀도 벽을 따라 미끄러진다.
- 바닥 높이는 `Staircase`가 (x, z)별로 알려주어, 안내방 평지·계단·메인방을
  걸을 때 시점이 자동으로 높낮이에 맞춰진다.

### 비고
- 목성계(`JupiterSystem`) 모듈은 팀원( 이경서 + 김기환)이 작성한 코드를 통합한 것이다.
- 앞으로 조명과 셰이더는 윤현정 조원이 담당해서 수정할 예정이다.
